# ER Diagrams

Use this reference when the model has reached entity-structure work: logical
entities, columns, indexes, constraints, generated foreign keys, `relate`
cardinality/options, and conceptual-to-logical mappings.

## Commands

```sh
# ER diagram — whole model
rdra-ish diagram src/ --kind er --format mermaid

# ER diagram — entities reachable from a BUC
rdra-ish diagram src/ --kind er --buc <BucId> --format mermaid

# Write Mermaid text to a specific file
rdra-ish diagram src/ --kind er --format mermaid --out docs/er

# Data contract exports
rdra-ish export src/ --kind dbml --out out/schema.dbml
rdra-ish export src/ --kind json-schema --out out/json-schema.json
```

## Reading The Output

- Each entity box lists columns with PK/FK markers.
- Column annotations such as indexes, composite uniques, checks, tenant/history,
  soft delete, derived columns, FK optionality, and on-delete/on-update actions are
  review signals; use list/export output for exact metadata.
- Crow's foot notation comes from `relate` predicates.
- FK columns are auto-generated by `relate`; do not add manual FK columns for the
  same relationship.

## Tips

- Scope with `--buc` when the whole ER diagram is too large for review.
- Pair ER output with `rdra-ish list src/ --kind entity --format json` when checking
  exact column metadata.
- Pair ER output with `rdra-ish csv src/ --kind api-matrix` when entity ownership is
  being reviewed through API boundaries.
- Pair ER output with conceptual `maps_to` review when business nouns are not all
  persisted as logical entities.
